    מילון מונחים¶

    כל מונח, שתי נקודות מבט. העמודה השמאלית היא כיצד קונה מדיה חושב על זה. העמודה הימנית היא כיצד מהנדס DevOps מוצא את זה במערכת.

    מונח הגדרה לקונה מדיה הגדרה DevOps / מערכת
    מושב (Seat) חשבון קונה ב-Google Authorized Buyers. אתם מגדירים את היקף הניתוח והטרגוט לפי מושב. buyer_account_id ב-Postgres. מאוחסן בטבלת seats. מסונכרן דרך GET /seats.
    QPS שאילתות לשנייה: הקצב המקסימלי של בקשות הצעת מחיר שאתם מבקשים מ-Google לשלוח. Google מגבילה את הנפח בפועל לפי דרגת החשבון שלכם. מגבלה מוגדרת לכל תצורת פרה-טרגוט. קצב כניסה בפועל מנוטר דרך מדדי משפך RTB ב-rtb_daily.
    בזבוז (Waste) QPS שנצרך על ידי בקשות הצעת מחיר שהמכרז שלכם דוחה (אזורים גאוגרפיים שגויים, גדלים שגויים, אין קריאייטיב תואם). כסף שהוצא על כלום. (total_qps - bids_placed) / total_qps. מחושב מצבירות rtb_daily. נראה ב-API של המשפך.
    תצורת פרה-טרגוט (Pretargeting config) הכללים ששולטים אילו בקשות הצעת מחיר מגיעות למכרז שלכם. מקבלים 10 לכל מושב. שולטת באזורים גאוגרפיים, גדלים, פורמטים, פלטפורמות, מפרסמים. ישות ניתנת לשינוי המסונכרנת מ-Google AB API. מאוחסנת ב-pretargeting_configs. מנוהלת דרך /settings/pretargeting. תמונות מצב מאפשרות חזרה לאחור.
    משפך (Funnel) ההתקדמות מבקשת הצעת מחיר להוצאה: QPS -> הצעות -> זכיות -> חשיפות -> קליקים -> הוצאה. בכל שלב יש נשירה. מחושב ממדדי rtb_daily. מוגש על ידי GET /analytics/rtb-funnel. צד הלקוח שומר במטמון ל-30 דקות.
    קריאייטיב (Creative) נכס מודעה: תמונה, וידאו, HTML או מקורי. יש לו פורמט, גודל, כתובת URL של יעד והיסטוריית ביצועים. שורה בטבלת creatives. תמונות ממוזערות באחסון blob. מסונכרן מ-Google AB API. ביצועים מצירופי rtb_daily.
    קמפיין (Campaign) קיבוץ לוגי של קריאייטיבים. משמש לארגון ניתוח ודיווח. שורה בטבלת ai_campaigns. קשר רבים-לרבים עם קריאייטיבים. תומך בקיבוץ אוטומטי בינה מלאכותית.
    כרטיס תצורה (Config card) פאנל ממשק המשתמש המציג את מצב תצורת פרה-טרגוט, QPS מקסימלי, אזורים גאוגרפיים, גדלים, פורמטים ופלטפורמות. רכיב React PretargetingConfigCard. נתונים מ-GET /settings/pretargeting-configs.
    רעננות נתונים (Data freshness) רשת המציגה אילו תאריכים מכילים נתונים מיובאים ("יובא") מול פערים ("חסר") לכל סוג דוח. GET /uploads/data-freshness. משתמש בשאילתות generate_series + EXISTS כנגד rtb_daily, rtb_bidstream, rtb_quality, rtb_bid_filtering. מגבלת זמן 30 שניות.
    ייבוא (Import) הכנסת נתוני ביצועים בפורמט CSV ל-Cat-Scan, בהעלאה ידנית או בייבוא אוטומטי מ-Gmail. CSV מנותח, מתוקף, עובר הסרת כפילויות (דרך אילוץ ייחודיות row_hash), ומוכנס לטבלאות היעד. העלאה בנתחים לקבצים מעל 5MB.
    חזרה לאחור (Rollback) החזרת שינוי בתצורת פרה-טרגוט למצב הקודם. תצוגה מקדימה עם הרצת ניסיון, ואז אישור. שחזור תמונת מצב: קורא מ-pretargeting_snapshots, מחיל הבדלים ל-Google AB API, רושם תמונת מצב חדשה. POST /snapshots/rollback.
    אופטימייזר / BYOM מערכת אוטומטית שמדרגת פלחים ומציעה שינויי תצורה. משתמשת במודל חיצוני שלכם. נקודת קצה לדירוג נקראת דרך HTTP POST. הצעות מאוחסנות ב-optimizer_proposals. מחזור חיים: דירוג -> הצעה -> אישור -> החלה.
    הגדרת תהליך עבודה (Workflow preset) זהיר, מאוזן או אגרסיבי. שולט ברמת הנועזות של הצעות האופטימייזר. פרמטר canary_profile ל-API של דירוג-והצעה. משפיע על ספי ביטחון ומגבלות היקף שינוי.
    CPM אפקטיבי (Effective CPM) מה שאתם משלמים בפועל לכל אלף חשיפות, בהתחשב בבזבוז ועלויות תשתית. מחושב ב-OptimizerEconomicsService. משלב נתוני הוצאה מ-rtb_daily עם עלות אירוח מוגדרת.
    המרה (Conversion) פעולת משתמש בעלת ערך (רכישה, הרשמה) שנרשמת לאחר חשיפה. מוזנת חזרה לאופטימיזציית טרגוט. אירוע נקלט דרך פיקסל (GET /conversions/pixel) או webhook (POST /conversions/webhook). מאוחסן בטבלאות המרות. מאומת באמצעות HMAC עבור webhooks.
    אחוז זכייה (Win rate) זכיות / הצעות. עד כמה ההצעות שלכם תחרותיות במכרז. auction_wins / bids_placed מ-rtb_daily.
    CTR קליקים / חשיפות. עד כמה הקריאייטיבים שלכם מעוררים מעורבות. clicks / impressions מ-rtb_daily.
    שער בדיקת תקינות בזמן ריצה (Runtime health gate) (לא מונח של קונה) תהליך CI v1-runtime-health-strict.yml. מריץ בדיקות מקצה לקצה: תקינות API, תקינות נתונים, המרות, אופטימייזר, SLO של QPS. מחזיר PASS/FAIL/BLOCKED לכל בדיקה.
    בדיקת חוזה (Contract check) (לא מונח של קונה) scripts/contracts_check.py. מאמת חוזי נתונים (כללים לא ניתנים למשא ומתן מייבוא ועד פלט API). רץ לאחר פריסה. חוסם שחרור בכשלון.
    Cloud SQL Proxy (לא מונח של קונה) קונטיינר נלווה המספק גישה מאומתת ל-Cloud SQL Postgres. חייב להיות תקין לפני שקונטיינר ה-API מתחיל.
    כותרת (לא מונח של קונה) כותרת HTTP שנקבעת על ידי OAuth2 Proxy לאחר אימות Google. נחשבת כמהימנה על ידי ה-API כאשר OAUTH2_PROXY_ENABLED=true. מוסרת על ידי nginx עבור בקשות חיצוניות.
