# Глава 1: Вход в систему

*Аудитория: все пользователи*

## Способы аутентификации

Cat-Scan поддерживает три способа входа:

| Способ | Как работает | Когда использовать |
|--------|--------------|--------------------|
| **Google OAuth** | Нажмите «Войти через Google» — произойдёт перенаправление через OAuth2 Proxy | Большинство пользователей. Используется аккаунт Google Workspace. |
| **Authing (OIDC)** | Нажмите «Войти через Authing» — произойдёт перенаправление к провайдеру OIDC | Организации, использующие Authing в качестве провайдера идентификации. |
| **Email и пароль** | Введите учётные данные непосредственно на странице входа | Локальные аккаунты, созданные администратором. |

## Первый вход

1. Перейдите по адресу `https://scan.rtb.cat` (или URL вашего развёртывания).
2. Вы увидите страницу входа с доступными вариантами авторизации.
3. Выберите способ и пройдите аутентификацию.
4. При первом входе система автоматически создаёт вашу учётную запись (для
   методов OAuth). Администратору, возможно, потребуется предоставить вам доступ
   к определённым местам покупателя (buyer seats).

## Переключатель мест (seat selector)

После входа вы увидите боковую панель с **переключателем мест** вверху.
Если у вашего аккаунта есть доступ к нескольким местам покупателя, используйте
выпадающий список для переключения между ними. Все данные на каждой странице
относятся к выбранному месту.

- **Одно место**: переключатель отображает название и ID вашего места.
- **Несколько мест**: выпадающий список позволяет переключаться. Каждая запись
  содержит отображаемое имя покупателя, `buyer_account_id` и количество
  креативов.
- **Кнопка «Sync All»**: обновляет креативы, эндпоинты и конфигурации
  претаргетинга из API Google для выбранного места.

## Если вход не удался

| Симптом | Вероятная причина | Что делать |
|---------|-------------------|------------|
| Цикл перенаправлений (страница постоянно перезагружается) | База данных недоступна, проверка авторизации завершается с ошибкой | Проверьте контейнер Cloud SQL Proxy. См. [Устранение неполадок](15-troubleshooting.md). |
| «Server unavailable» (502/503/504) | Контейнер API или nginx не работает | Обратитесь к команде DevOps. См. [Мониторинг состояния](13-health-monitoring.md). |
| «Authentication required» | Сессия истекла или cookie удалены | Войдите в систему повторно. |
| «You don't have access to this buyer account» | Права доступа к этому месту не предоставлены | Обратитесь к администратору. См. [Управление пользователями](16-user-admin.md). |
