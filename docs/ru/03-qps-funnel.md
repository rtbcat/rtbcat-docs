# Глава 3: Понимание воронки QPS

*Аудитория: медиабайеры, менеджеры кампаний*

Это главная страница Cat-Scan (`/`). Всё начинается здесь.

## Что вы видите

Страница QPS Waste Optimizer отображает воронку RTB (путь от запроса на ставку
до расхода) и показывает, где происходит падение объёма.

![Главная страница QPS Waste Optimizer](images/screenshot-qps-home.png)

### Воронка

| Этап | Что означает |
|------|--------------|
| **QPS** | Максимальное количество запросов на ставку в секунду, которое вы запрашиваете у Google. Google регулирует фактический объём в зависимости от уровня аккаунта, поэтому обычно вы получаете меньше установленного лимита. |
| **Ставки (Bids)** | Сколько из этих запросов ваш биддер решил участвовать. Остальные были отклонены (неподходящий инвентарь, нет соответствующего креатива, цена ниже порога). |
| **Победы (Wins)** | Аукционы, которые ваш биддер выиграл. Вы платите только за победы. |
| **Показы (Impressions)** | Объявления, фактически показанные пользователям после победы в аукционе. |
| **Клики (Clicks)** | Взаимодействия пользователей с показанными объявлениями. |
| **Расход (Spend)** | Общая сумма, потраченная на выигранные показы. |

Разрыв между этапами — это пространство для оптимизации. Большое падение от QPS
к ставкам означает, что биддер отклоняет большую часть того, что отправляет
Google — типичные потери, которые можно устранить претаргетингом.

### Ключевые метрики

- **Win rate (процент побед)**: Wins / Bids. Насколько конкурентоспособны ваши ставки.
- **CTR (кликабельность)**: Clicks / Impressions. Насколько привлекательны ваши креативы.
- **CPM (стоимость за тысячу показов)**: Cost per thousand impressions. Сколько вы платите за видимость.
- **Waste ratio (доля потерь)**: (QPS - Bids) / QPS. Доля трафика, который вы не можете использовать.

### Карточки конфигураций претаргетинга

Под воронкой отображаются карточки каждой из ваших конфигураций претаргетинга
(до 10 на место). Каждая карточка содержит:

- **Состояние**: Active (активна) или Suspended (приостановлена)
- **Максимальный QPS**: лимит запросов на ставку для данной конфигурации
- **Форматы**: VIDEO, DISPLAY_IMAGE, DISPLAY_HTML, NATIVE
- **Платформы**: DESKTOP, MOBILE_APP, MOBILE_WEB, CONNECTED_TV
- **Географии**: включённые и исключённые географические цели
- **Размеры**: включённые размеры объявлений (или все, если фильтр не установлен)

### Элементы управления

- **Выбор периода**: 7, 14 или 30 дней данных
- **Фильтр по месту**: ограничение данных конкретным местом покупателя
- **Переключатель конфигурации**: детализация по конкретной конфигурации претаргетинга

## Как читать данные

Начните с доли потерь (waste ratio). Если она выше 50%, у вас значительный
потенциал для улучшения. Затем посмотрите, какие конфигурации вносят наибольший
вклад в потери. Перейдите к детализации по измерениям
([География](04-analyzing-waste.md), [Издатели](04-analyzing-waste.md),
[Размеры](04-analyzing-waste.md)), чтобы найти конкретные источники.
