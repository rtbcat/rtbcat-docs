# Глава 6: Настройка претаргетинга

*Аудитория: медиабайеры, менеджеры рекламных кампаний*

Конфигурации претаргетинга — ваш основной инструмент управления тем, что Google
отправляет вашему бидеру. В этой главе описано, как безопасно управлять ими
в Cat-Scan.

## Что контролирует конфигурация претаргетинга

Каждая конфигурация — это набор правил, которые говорят Google: «присылай мне
только те bid-запросы, которые соответствуют этим критериям». На каждый сит
выделяется **10 конфигураций**.

| Поле | Что фильтрует |
|------|--------------|
| **Состояние** | Active (получает трафик) или Suspended (приостановлен). |
| **Макс. QPS** | Верхний предел количества запросов в секунду, принимаемых этой конфигурацией. |
| **Гео (включённые)** | Страны, регионы или города, из которых принимается трафик. |
| **Гео (исключённые)** | Географии, которые блокируются, даже если совпадают с включениями. |
| **Размеры (включённые)** | Принимаемые размеры объявлений (например, 300x250, 728x90). |
| **Форматы** | Типы креативов: VIDEO, DISPLAY_IMAGE, DISPLAY_HTML, NATIVE. |
| **Платформы** | Типы устройств: DESKTOP, MOBILE_APP, MOBILE_WEB, CONNECTED_TV. |
| **Паблишеры** | Списки разрешённых/запрещённых доменов или приложений издателей. |

## Чтение карточки конфигурации

На главной странице и в настройках каждая конфигурация отображается в виде
карточки с текущим состоянием.

![Карточки конфигураций претаргетинга с активным и приостановленным состояниями](images/screenshot-pretargeting-configs.png)

На что обращать внимание:

- **Active + высокий макс. QPS + широкие гео** — эта конфигурация принимает много
  трафика. Если при этом она имеет высокий уровень потерь, это ваша главная
  цель для оптимизации.
- **Suspended** — не получает трафик. Полезно для подготовки изменений перед
  запуском.
- **Included sizes: (all)** — принимаются все размеры объявлений, которые
  присылает Google. Для фиксированных размеров дисплейной рекламы это почти
  наверняка расточительно.

## Внесение изменений

### Процесс с предварительным просмотром (dry-run)

1. Перейдите к конфигурации, которую хотите изменить (главная страница или
   `/settings/system`).
2. Выберите поле для изменения (например, исключённые гео, включённые размеры).
3. Введите новые значения.
4. Нажмите **Preview** (dry-run). Cat-Scan покажет, что именно изменится,
   без фактического применения.
5. Если предварительный просмотр выглядит корректно, нажмите **Apply**.
6. Изменение фиксируется в истории с меткой времени и вашей идентификацией.

### Редактор разрешений/запретов для паблишеров

Для блокировки на уровне издателей Cat-Scan предоставляет отдельный редактор
для каждой конфигурации. Вы можете:
- Искать паблишеров по имени домена
- Блокировать отдельные домены или приложения
- Разрешать конкретные домены, переопределяя более широкие блокировки
- Применять изменения массово

Это значительно проще, чем управление паблишерами через интерфейс Authorized
Buyers.

## История изменений (`/history`)

Каждое изменение претаргетинга фиксируется в хронологии на странице `/history`.

![Хронология изменений с фильтрами и экспортом](images/screenshot-change-history.png)

Для каждой записи отображается:
- **Когда**: метка времени изменения
- **Кто**: пользователь, выполнивший изменение
- **Что**: имя поля, старое значение, новое значение
- **Тип**: вид изменения (добавление, удаление, обновление)

## Откат

Если изменение привело к проблемам (например, выросли потери, упал win rate),
вы можете его откатить:

1. Перейдите на `/history`.
2. Найдите изменение, которое хотите отменить.
3. Нажмите **Preview rollback**. Откроется предварительный просмотр отката
   к предыдущему состоянию.
4. При необходимости укажите причину отката.
5. Нажмите **Confirm rollback**.

Сам откат фиксируется как новая запись в истории, обеспечивая полный аудиторский
след.
