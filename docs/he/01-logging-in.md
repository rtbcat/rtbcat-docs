# פרק 1: התחברות

*קהל יעד: כולם*

## שיטות אימות

Cat-Scan תומכת בשלוש שיטות התחברות:

| שיטה | כיצד פועלת | מתי להשתמש |
|--------|-------------|-------------|
| **Google OAuth** | לחצו על "Sign in with Google", שמפנה דרך OAuth2 Proxy | רוב המשתמשים. משתמשת בחשבון Google Workspace שלכם. |
| **Authing (OIDC)** | לחצו על "Sign in with Authing", שמפנה לספק OIDC | ארגונים שמשתמשים ב-Authing כספק הזהות שלהם. |
| **דוא"ל וסיסמה** | הזינו פרטי התחברות ישירות בדף ההתחברות | חשבונות מקומיים שנוצרו על ידי מנהל מערכת. |

## התחברות ראשונה

1. נווטו אל `https://scan.rtb.cat` (או כתובת ה-URL של הפריסה שלכם).
2. יוצג לכם דף ההתחברות עם אפשרויות הכניסה הזמינות.
3. בחרו את השיטה שלכם והזדהו.
4. בהתחברות הראשונה, המערכת יוצרת את רשומת המשתמש שלכם אוטומטית (עבור שיטות
   OAuth). ייתכן שמנהל המערכת יצטרך להעניק לכם גישה לחשבונות קונה ספציפיים.

## בורר החשבונות

לאחר ההתחברות, תראו את סרגל הצד עם **בורר החשבונות** בחלקו העליון. אם לחשבונכם
יש גישה למספר חשבונות קונה, השתמשו בתפריט הנפתח כדי לעבור ביניהם. כל הנתונים
בכל עמוד מוגבלים לחשבון הנבחר.

- **חשבון יחיד**: הבורר מציג את שם החשבון וה-ID ישירות.
- **מספר חשבונות**: תפריט נפתח מאפשר מעבר. כל רשומה מציגה את שם התצוגה של
  הקונה, את ה-`buyer_account_id` ומספר קריאייטיבים.
- **כפתור "Sync All"**: מרענן קריאייטיבים, נקודות קצה ותצורות pretargeting
  מה-API של גוגל עבור החשבון הנבחר.

## כאשר ההתחברות נכשלת

| תסמין | סיבה סבירה | מה לעשות |
|---------|-------------|------------|
| לולאת הפניות (הדף נטען שוב ושוב) | בסיס הנתונים אינו נגיש, כך שבדיקת האימות נכשלת בשקט | בדקו את קונטיינר Cloud SQL Proxy. ראו [פתרון בעיות](15-troubleshooting.md). |
| "Server unavailable" (502/503/504) | קונטיינר API או nginx מושבת | פנו לצוות ה-DevOps שלכם. ראו [ניטור תקינות](13-health-monitoring.md). |
| "Authentication required" | הפגישה פגה או העוגיה נמחקה | התחברו מחדש. |
| "You don't have access to this buyer account" | לא הוענקו הרשאות לחשבון זה | פנו למנהל המערכת. ראו [ניהול משתמשים](16-user-admin.md). |
