# פרק 9: ייבוא נתונים

*קהל יעד: קונים מדיה, מנהלי קמפיינים*

הניתוח של Cat-Scan תלוי לחלוטין בנתוני ביצועים מ-Google Authorized
Buyers. מאחר ש-Google אינה מספקת API לדיווח, כל הנתונים מגיעים מייצוא
CSV. פרק זה מסביר כיצד להכניס נתונים ל-Cat-Scan וכיצד לוודא
שהם מגיעים.

## למה זה חשוב

ללא נתונים מיובאים, ל-Cat-Scan אין מה לנתח. המשפך, תצוגות הבזבוז,
ביצועי קריאייטיב והאופטימייזר כולם תלויים בנתוני CSV עדכניים. אם
הנתונים שלכם מיושנים, ההחלטות שלכם מבוססות על מידע ישן.

## שתי דרכים להגעת נתונים

### 1. העלאת CSV ידנית (`/import`)

גררו ושחררו קובץ CSV שיוצא מ-Google Authorized Buyers.

![דף ייבוא נתונים עם אזור העלאה ורשת רעננות](images/screenshot-import.png)

**תהליך העבודה:**

1. ייצאו את הדוח מחשבון Google Authorized Buyers שלכם.
2. עברו ל-`/import` ב-Cat-Scan.
3. גררו את הקובץ לאזור השחרור (או לחצו לעיון).
4. Cat-Scan **מזהה אוטומטית את סוג הדוח** ומציגה תצוגה מקדימה:
   - עמודות נדרשות מול עמודות שנמצאו
   - ספירת שורות וטווח תאריכים
   - שגיאות אימות אם ישנן
5. בדקו את התצוגה המקדימה. אם עמודות דורשות מיפוי מחדש, השתמשו בעורך
   מיפוי העמודות.
6. לחצו על **ייבוא**.
7. סרגל התקדמות מציג את סטטוס ההעלאה. קבצים מעל 5MB מועלים בחלקים
   באופן אוטומטי.
8. התוצאות מציגות: שורות שיובאו, כפילויות שדולגו, שגיאות אם ישנן.

**סוגי דוחות** שמזוהים אוטומטית:

| סוג | תבנית שם CSV | מה הוא מכיל |
|------|-----------------|------------------|
| bidsinauction | `catscan-report-*` | ביצועי RTB יומיים: חשיפות, הצעות מחיר, זכיות, הוצאה |
| quality | `catscan-report-*` (מדדי איכות) | אותות איכות: נצפות, הונאה, בטיחות מותג |
| pipeline-geo | `*-pipeline-geo-*` | פילוח גיאוגרפי של זרם ההצעות |
| pipeline-publisher | `*-pipeline-publisher-*` | פילוח לפי דומיין מפרסם |
| bid-filtering | `*-bid-filtering-*` | סיבות וכמויות סינון הצעות מחיר |

### 2. ייבוא אוטומטי מ-Gmail

Cat-Scan יכול לקלוט דוחות אוטומטית מחשבון Gmail מחובר.

- Google Authorized Buyers שולח דוחות יומיים באימייל.
- אינטגרציית ה-Gmail של Cat-Scan קוראת את האימיילים האלה ומייבאת את קבצי
  ה-CSV המצורפים באופן אוטומטי.
- בדקו סטטוס ב-`/settings/accounts` > לשונית דוחות Gmail, או דרך
  `/gmail/status` ב-API.

**לאימות שייבוא Gmail עובד:**
- בדקו את פאנל סטטוס Gmail: `last_reason` צריך להיות `running`.
- בדקו את ספירת `unread`: מספר גדול של אימיילים שלא נקראו עשוי להצביע שהייבוא
  תקוע.
- בדקו את היסטוריית הייבוא לרשומות אחרונות.

## רשת רעננות נתונים

רשת רעננות הנתונים (נראית ב-`/import` ומשמשת את שער בריאות זמן
הריצה) מציגה **מטריצת תאריך x סוג-דוח**:

```
              bidsinauction   quality   pipeline-geo   pipeline-publisher   bid-filtering
2026-03-02    imported        missing   imported       imported             imported
2026-03-01    imported        missing   imported       imported             imported
2026-02-28    imported        imported  imported       imported             imported
...
```

- **imported**: ל-Cat-Scan יש נתונים עבור תאריך וסוג דוח זה.
- **missing**: לא נמצאו נתונים. הדוח לא יוצא, לא התקבל
  ב-Gmail, או שהייבוא נכשל.

**אחוז כיסוי** מסכם את מידת השלמות של הנתונים שלכם לאורך
חלון המבט לאחור. שער בריאות זמן הריצה משתמש בזה כדי לקבוע אם המערכת
פעילה.

## ניקוי כפילויות

ייבוא מחדש של אותו CSV (או ש-Gmail מעבד מחדש את אותו אימייל) **לא**
גורם לספירה כפולה של נתונים. כל שורה עוברת חישוב גיבוב, וכפילויות מדולגות
בעת ההכנסה. המשמעות היא שתמיד בטוח לייבא מחדש.

## היסטוריית ייבוא

טבלת היסטוריית הייבוא ב-`/import` מציגה את 20 הייבואים האחרונים:

- חותמת זמן
- שם קובץ
- ספירת שורות
- גורם הייבוא (העלאה ידנית לעומת gmail-auto)
- סטטוס (הושלם, נכשל, כפילות)

## פתרון בעיות

| בעיה | מה לבדוק |
|---------|---------------|
| תאים "חסרים" ברשת רעננות | האם הדוח יוצא מ-Google בתאריך זה? בדקו ב-Gmail את האימייל. |
| ייבוא נכשל עם שגיאת אימות | חוסר התאמה בעמודות. בדקו את טבלת העמודות הנדרשות מול ה-CSV שלכם. |
| ייבוא Gmail מציג "stopped" | בדקו ב-`/settings/accounts` > לשונית Gmail. ייתכן שצריך להפעיל מחדש או לאשר מחדש. |
| אחוז כיסוי יורד | דוחות מגיעים אך עבור פחות תאריכים מהצפוי. בדקו את לוח הזמנים של הייצוא ב-Google AB. |
