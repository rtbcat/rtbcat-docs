# Розділ 1: Вхід до системи

*Аудиторія: всі*

## Методи автентифікації

Cat-Scan підтримує три методи входу:

| Метод | Як це працює | Коли використовувати |
|-------|-------------|----------------------|
| **Google OAuth** | Натисніть «Sign in with Google» — вас буде перенаправлено через OAuth2 Proxy | Більшість користувачів. Використовує ваш обліковий запис Google Workspace. |
| **Authing (OIDC)** | Натисніть «Sign in with Authing» — вас буде перенаправлено до провайдера OIDC | Організації, що використовують Authing як свого провайдера ідентифікації. |
| **Електронна пошта та пароль** | Введіть облікові дані безпосередньо на сторінці входу | Локальні облікові записи, створені адміністратором. |

## Перший вхід

1. Перейдіть за адресою `https://scan.rtb.cat` (або URL вашого розгортання).
2. Ви побачите сторінку входу з доступними варіантами автентифікації.
3. Виберіть свій метод і пройдіть автентифікацію.
4. При першому вході система автоматично створює ваш обліковий запис
   користувача (для методів OAuth). Вашому адміністратору може знадобитися
   надати вам доступ до конкретних seat-ів покупця.

## Вибір seat-у

Після входу ви побачите бічну панель із **вибором seat-у** у верхній частині.
Якщо ваш обліковий запис має доступ до кількох seat-ів покупця, використовуйте
випадаючий список для перемикання між ними. Усі дані на кожній сторінці
прив'язані до обраного seat-у.

- **Один seat**: селектор показує назву вашого seat-у та ID безпосередньо.
- **Кілька seat-ів**: випадаючий список дає змогу перемикатися. Кожен запис
  показує відображувану назву покупця, `buyer_account_id` та кількість
  креативів.
- **Кнопка «Sync All»**: оновлює креативи, endpoint-и та конфігурації
  претаргетингу з API Google для обраного seat-у.

## Коли вхід не вдається

| Симптом | Ймовірна причина | Що робити |
|---------|-----------------|-----------|
| Цикл перенаправлень (сторінка постійно перезавантажується) | База даних недоступна, тому перевірка автентифікації мовчки завершується невдачею | Перевірте контейнер Cloud SQL Proxy. Див. [Усунення неполадок](15-troubleshooting.md). |
| «Server unavailable» (502/503/504) | Контейнер API або nginx не працює | Зверніться до вашої команди DevOps. Див. [Моніторинг стану](13-health-monitoring.md). |
| «Authentication required» | Сесія закінчилася або cookie було видалено | Увійдіть знову. |
| «You don't have access to this buyer account» | Дозволи для цього seat-у не надані | Зверніться до свого адміністратора. Див. [Адміністрування користувачів](16-user-admin.md). |
