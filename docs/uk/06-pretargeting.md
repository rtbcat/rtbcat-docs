# Розділ 6: Налаштування претаргетингу

*Аудиторія: медіабаєри, менеджери кампаній*

Конфігурації претаргетингу — це ваш основний інструмент для контролю того, що
Google надсилає вашому бідеру. Цей розділ описує, як безпечно керувати ними
в Cat-Scan.

## Що контролює конфігурація претаргетингу

Кожна конфігурація — це набір правил, які повідомляють Google: «надсилайте мені
лише ті bid-запити, що відповідають цим критеріям». Ви отримуєте **10 конфігурацій
на місце (seat)**.

| Поле | Що фільтрує |
|------|-------------|
| **Стан** | Активний (отримує трафік) або Призупинений (пауза). |
| **Максимальний QPS** | Верхня межа запитів на секунду, яку приймає ця конфігурація. |
| **Геолокації (включені)** | Країни, регіони або міста, з яких отримувати трафік. |
| **Геолокації (виключені)** | Географії для блокування, навіть якщо вони відповідають включенням. |
| **Розміри (включені)** | Розміри оголошень для прийому (наприклад, 300x250, 728x90). |
| **Формати** | Типи креативів: VIDEO, DISPLAY_IMAGE, DISPLAY_HTML, NATIVE. |
| **Платформи** | Типи пристроїв: DESKTOP, MOBILE_APP, MOBILE_WEB, CONNECTED_TV. |
| **Видавці** | Списки дозволених/заблокованих доменів або додатків видавців. |

## Читання картки конфігурації

На головній сторінці та в налаштуваннях кожна конфігурація відображається
як картка з її поточним станом.

![Картки конфігурації претаргетингу з активним та призупиненим станами](images/screenshot-pretargeting-configs.png)

Основні моменти, на які варто звернути увагу:

- **Активний + високий максимальний QPS + широкі геолокації** — ця конфігурація
  перехоплює багато трафіку. Якщо вона також має високий рівень витрат, це ваша
  найбільша ціль для оптимізації.
- **Призупинений** — не отримує трафік. Корисно для підготовки змін перед
  запуском.
- **Включені розміри: (всі)** — приймає кожний розмір оголошення, який надсилає
  Google. Для дисплейної реклами з фіксованим розміром це майже напевно
  марнотратно.

## Внесення змін

### Робочий процес із пробним запуском

1. Перейдіть до конфігурації, яку хочете змінити (головна сторінка або
   `/settings/system`).
2. Виберіть поле для зміни (наприклад, виключені геолокації, включені розміри).
3. Введіть нові значення.
4. Натисніть **Попередній перегляд** (пробний запуск). Cat-Scan покаже вам
   точно, що зміниться, без застосування.
5. Якщо попередній перегляд виглядає правильно, натисніть **Застосувати**.
6. Зміна записується в історію з міткою часу та вашою ідентифікацією.

### Редактор дозволів/заборон видавців

Для блокування на рівні видавців Cat-Scan надає спеціальний редактор
для кожної конфігурації. Ви можете:
- Шукати видавців за доменним іменем
- Блокувати окремі домени або додатки
- Дозволяти конкретні домени, які перевизначають ширші блокування
- Застосовувати зміни масово

Це значно простіше, ніж керування видавцями через інтерфейс Authorized
Buyers.

## Історія змін (`/history`)

Кожна зміна претаргетингу записується в хронологію на `/history`.

![Хронологія змін із фільтрами та експортом](images/screenshot-change-history.png)

Для кожного запису ви бачите:
- **Коли**: мітка часу зміни
- **Хто**: користувач, який її здійснив
- **Що**: назва поля, старе значення, нове значення
- **Тип**: вид зміни (додавання, видалення, оновлення)

## Відкат

Якщо зміна спричинює проблеми (наприклад, зростання витрат, падіння
win rate), ви можете її відкотити:

1. Перейдіть на `/history`.
2. Знайдіть зміну, яку хочете скасувати.
3. Натисніть **Попередній перегляд відкату**. Це покаже пробний запуск
   повернення до попереднього стану.
4. За бажанням додайте причину відкату.
5. Натисніть **Підтвердити відкат**.

Сам відкат записується як новий запис в історії, тому у вас є
повний аудиторський слід.
