# 第 0 章：什么是 Cat-Scan？

*适用读者：所有人*

Cat-Scan 是一个面向 Google Authorized Buyers 的 QPS 优化平台。它让你了解
竞价器的每秒查询量（QPS）分配的使用情况（以及浪费情况），并提供改进工具。

![QPS 漏斗](../assets/qps-funnel.svg)

## 核心问题

当你在 Google Authorized Buyers 交易平台上运营一个席位时，Google 会向你的
竞价器端点发送竞价请求流。你为此付费：它消耗你分配的 QPS、竞价器的算力
和网络带宽。

但并非每个竞价请求都有用。许多请求对应的是你永远不会购买的库存：你不定向
的国家、你从未听说过的发布商、没有匹配素材的广告尺寸。你的竞价器仍然需要
接收并拒绝每一条请求。

在典型场景中，**超过一半的 QPS 是浪费的。**

## Cat-Scan 如何解决

Cat-Scan 与你的竞价器配合工作，提供三方面能力：

### 1. 可见性

它从 Google 的 CSV 导出文件重建效果报告（因为没有报告 API），展示完整的
RTB 漏斗：从原始 QPS 到出价、胜出、展示、点击和花费。可按任意维度拆分：
地理、发布商、广告尺寸、素材、预定向配置。

这让你能回答以下问题：
- 哪些国家消耗了 QPS 却没有产生胜出？
- 哪些发布商 QPS 很高但花费为零？
- 哪些广告尺寸收到了流量但没有匹配素材？
- 哪些预定向配置表现好、哪些表现差？

### 2. 控制力

Google 给你每个席位 10 个预定向配置。这些是你告诉 Google 发送什么流量、
过滤什么流量的主要杠杆。Cat-Scan 提供：
- 带试运行预览的配置编辑器
- 带一键回滚的变更历史时间线
- 每个配置的发布商允许/拒绝列表
- 对细分市场评分并提出配置变更建议的优化器

### 3. 安全性

每个预定向变更都被记录。你可以在应用前预览变更效果。如果出了问题，可以
立即回滚。优化器使用工作流预设（保守、平衡、激进），确保未经人工审核的
自动化变更不会生效。

## 关键概念

继续之前，请确保理解以下术语：

| 概念 | 含义 |
|------|------|
| **席位** | Google Authorized Buyers 上的买方账户，通过 `buyer_account_id` 标识。一个组织可以有多个席位。 |
| **QPS** | 每秒查询量（Queries Per Second），即你要求 Google 发送的最大竞价请求速率。Google 会根据你的账户等级进行节流，因此实际收到的量通常低于上限，需要高效使用。 |
| **预定向** | 服务端过滤器，告诉 Google 向你发送哪些竞价请求。控制：地理位置、广告尺寸、格式、平台。每个席位有 10 个。 |
| **RTB 漏斗** | 从接收竞价请求到花费的过程：QPS -> 出价 -> 胜出 -> 展示 -> 点击 -> 转化。每一步都有流失；Cat-Scan 展示流失发生的位置。 |
| **浪费** | 竞价器无法或不愿使用的竞价请求所消耗的 QPS。目标是在不失去有价值流量的前提下减少浪费。 |
| **配置** | 预定向配置的简称。每个配置有状态（激活/暂停）、最大 QPS，以及地区、尺寸、格式和平台的包含/排除规则。 |
