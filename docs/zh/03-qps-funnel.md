# 第 3 章：理解你的 QPS 漏斗

*适用读者：媒体买手、投放经理*

这是 Cat-Scan 的首页（`/`）。一切从这里开始。

## 你看到什么

QPS 浪费优化器页面展示你的 RTB 漏斗（从竞价请求到花费的过程），并高亮
流量流失的位置。

![QPS 浪费优化器首页](../images/screenshot-qps-home.png)

### 漏斗

| 阶段 | 含义 |
|------|------|
| **QPS** | 你要求 Google 每秒发送的最大竞价请求数。Google 会根据账户等级节流，因此实际收到的量通常低于上限。 |
| **出价** | 其中你的竞价器选择出价的请求数。剩余的被拒绝（不匹配的库存、没有对应素材、低于底价）。 |
| **胜出** | 你的竞价器赢得的拍卖数。你只为胜出付费。 |
| **展示** | 胜出后实际向用户展示的广告数。 |
| **点击** | 用户与展示广告的互动数。 |
| **花费** | 在胜出展示上的总花费。 |

每个阶段之间的落差就是优化机会所在。QPS 到出价的大幅下降意味着你的竞价器
拒绝了 Google 发送的大部分内容，这是预定向可以解决的典型浪费。

### 关键指标

- **胜出率**：胜出 / 出价。衡量你的出价竞争力。
- **CTR**：点击 / 展示。衡量你的素材吸引力。
- **CPM**：每千次展示成本。你为可见度支付的费用。
- **浪费比率**：(QPS - 出价) / QPS。无法使用的流量占比。

### 预定向配置卡片

漏斗下方是每个预定向配置的卡片（每个席位最多 10 个）。每张卡片显示：

- **状态**：激活或暂停
- **最大 QPS**：此配置接受的竞价请求上限
- **格式**：VIDEO、DISPLAY_IMAGE、DISPLAY_HTML、NATIVE
- **平台**：DESKTOP、MOBILE_APP、MOBILE_WEB、CONNECTED_TV
- **地区**：包含和排除的地理目标
- **尺寸**：包含的广告尺寸（未过滤则为全部）

### 控制项

- **时间段选择器**：7、14 或 30 天数据
- **席位筛选器**：限定到特定买方席位
- **配置切换**：深入查看特定预定向配置

## 如何解读

从浪费比率开始。如果超过 50%，说明有很大的改进空间。然后查看哪些配置
贡献了最多浪费。点击进入维度分析（[地理](04-analyzing-waste.md)、
[发布商](04-analyzing-waste.md)、[尺寸](04-analyzing-waste.md)）找到
具体来源。
