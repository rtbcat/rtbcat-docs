# 第 8 章：转化与归因

*适用读者：媒体买手、投放经理*

转化追踪让 Cat-Scan 衡量展示后发生了什么：用户是否采取了有价值的行动？
这些数据输入到优化器的评分中，帮助你评估真实的活动效果。

## 转化来源

Cat-Scan 支持两种集成方式：

### 像素

追踪像素在你的转化页面（如结账确认页）触发。

- 端点：`/api/conversions/pixel`
- 参数：`buyer_id`、`source_type=pixel`、`event_name`、`event_value`、
  `currency`、`event_ts`
- 除了在页面上放置像素外，无需服务端设置。

### Webhook

你的服务器向 Cat-Scan 的 Webhook 端点发送转化事件。

- 比像素更可靠（无广告拦截器、无客户端依赖）。
- 需要服务端集成。
- 支持 HMAC 签名验证以确保安全。

## Webhook 安全

Cat-Scan 提供分层 Webhook 安全：

| 功能 | 作用 |
|------|------|
| **HMAC 验证** | 每个 Webhook 请求使用共享密钥签名。Cat-Scan 拒绝未签名或签名错误的请求。 |
| **速率限制** | 通过限制每个时间窗口的请求数量防止滥用。 |
| **新鲜度监控** | 当 Webhook 事件停止到达时发出告警（过期检测）。 |

在 `/settings/system` > 转化健康中配置 Webhook 安全。

## 就绪检查

在依赖转化数据之前，验证就绪状态：

1. 前往 `/settings/system` 或引导清单。
2. 检查**转化就绪状态**：显示来源是否已连接并在预期新鲜度窗口内
   交付事件。
3. 检查**摄入统计**：按来源类型和时间段的事件计数。

## 转化健康

转化健康面板显示：

- 摄入状态（是否正在接收事件）
- 聚合状态（事件是否正在被处理为指标）
- 最后一次事件时间戳
- 错误计数（如有）
