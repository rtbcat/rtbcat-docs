# 第 1 章：登录

*适用读者：所有人*

## 认证方式

Cat-Scan 支持三种登录方式：

| 方式 | 工作原理 | 使用场景 |
|------|---------|---------|
| **Google OAuth** | 点击"使用 Google 登录"，通过 OAuth2 Proxy 重定向 | 大多数用户。使用你的 Google Workspace 账户。 |
| **Authing（OIDC）** | 点击"使用 Authing 登录"，重定向到 OIDC 提供商 | 使用 Authing 作为身份提供商的组织。 |
| **邮箱与密码** | 在登录页面直接输入凭证 | 由管理员创建的本地账户。 |

## 首次登录

1. 导航到 `https://scan.rtb.cat`（或你的部署地址）。
2. 你会看到登录页面及可用的登录选项。
3. 选择登录方式并完成认证。
4. 首次登录时，系统会自动创建你的用户记录（OAuth 方式）。你的管理员可能
   需要授予你访问特定买方席位的权限。

## 席位选择器

登录后，你会在侧边栏顶部看到**席位选择器**。如果你的账户有多个买方席位的
访问权限，使用下拉菜单切换。所有页面的数据都限定在所选席位范围内。

- **单个席位**：选择器直接显示你的席位名称和 ID。
- **多个席位**：下拉菜单可切换。每个条目显示买方名称、`buyer_account_id`
  和素材数量。
- **"全部同步"按钮**：从 Google API 刷新所选席位的素材、端点和预定向配置。

## 登录失败

| 症状 | 可能原因 | 处理方法 |
|------|---------|---------|
| 重定向循环（页面不断刷新） | 数据库不可达，认证检查静默失败 | 检查 Cloud SQL Proxy 容器。参见[故障排除](15-troubleshooting.md)。 |
| "服务器不可用"（502/503/504） | API 或 nginx 容器宕机 | 联系运维团队。参见[健康监控](13-health-monitoring.md)。 |
| "需要认证" | 会话过期或 Cookie 被清除 | 重新登录。 |
| "你没有此买方账户的访问权限" | 未授予此席位的权限 | 联系管理员。参见[用户管理](16-user-admin.md)。 |
